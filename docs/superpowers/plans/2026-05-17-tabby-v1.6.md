# Tabby v1.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hard-coded 6-hour consolidate cooldown with a popup-configurable preset (default 5 min), add growth-aware re-arm so users on long cooldowns still get pitched on tab explosions, and extend the consolidate interstitial to show the list of existing host tabs (click to focus) â€” fixing the "I have 7 ebay tabs and Tabby never speaks" failure mode.

**Architecture:** Single Chrome MV3 extension. Pure cooldown logic lives in `src/background.js` as a testable function (`evaluateConsolidationPitch`). Settings persist in `chrome.storage.local` under one `settings` key. Per-host pitch records persist in `chrome.storage.session` and grow from a bare number to `{ at, count }` â€” legacy-compatible read path. Interstitial reuses the duplicate-mode list-row pattern in a new `#con-list` element.

**Tech Stack:** Vanilla JS (no bundler), Chrome MV3 APIs (`storage.local`, `storage.session`, `tabs`, `webNavigation`, `sessions`), Node built-in test runner (`node:test`, `node:assert`) for the one pure-function test.

**Spec:** `docs/superpowers/specs/2026-05-17-tabby-v1.6-design.md` (read this first if anything below is ambiguous).

**File map:**
- Modify: `manifest.json` (version bump only)
- Create: `src/cooldown.mjs` (pure cooldown decision function + constants, no chrome deps)
- Modify: `src/background.js` (import cooldown, add `getSettings`, hostTabs, payload shape)
- Modify: `src/popup.html` (add `<select>`)
- Modify: `src/popup.js` (load/save settings)
- Modify: `src/interstitial.html` (add `#con-list`)
- Modify: `src/interstitial.css` (list scroll behavior in consolidate panel â€” minor)
- Modify: `src/interstitial.js` (render hostTabs, wire row click)
- Create: `tests/cooldown.test.mjs` (pure-function test)

**Why a separate `cooldown.mjs`:** Node's test runner needs ESM. Tabby has no `package.json` (so Node treats `.js` files as CommonJS) and adding one risks side-effects on existing scripts (`scripts/build-signals.cjs`, etc.). Putting the pure logic in `cooldown.mjs` makes it unambiguously an ES module for both Node and the Chrome MV3 service worker (which is already loaded as `"type": "module"` per the manifest).

```javascript
/**
 * Pure consolidation-pitch decision logic. No chrome.* dependencies.
 *
 * Imported by:
 *   - src/background.js  (the MV3 service worker)
 *   - tests/cooldown.test.mjs  (Node built-in test runner)
 *
 * Living here as a separate ES module keeps the logic testable in Node
 * (which would otherwise treat the SW file as CommonJS without a
 * package.json "type": "module" declaration).
 */

/**
 * Default cooldown when no user setting exists yet (5 min).
 * Tuned for short bursts of tab sprawl rather than long quiet sessions.
 */
export const DEFAULT_COOLDOWN_MS = 5 * 60 * 1000;

/**
 * Re-arm growth threshold: re-pitch a host once its tab count grows by this
 * much since the last pitch, even mid-cooldown. Tunable here, not in user
 * settings â€” keeps the configuration surface small.
 */
export const REARM_GROWTH = 2;

/**
 * `prior` accepts both the new `{ at, count }` shape and the legacy bare
 * `number` (timestamp only) so v1.5 session data doesn't break v1.6 on first
 * load after upgrade.
 *
 * Returns true if Tabby should pitch the consolidate interstitial now.
 *
 * Rules:
 *  - No prior pitch â†’ always allowed.
 *  - cooldownMs === 0 â†’ "Off" mode. Time check disabled; only growth can re-arm.
 *  - Otherwise: fire if cooldown elapsed OR tab count grew by REARM_GROWTH+.
 */
export function evaluateConsolidationPitch({ prior, currentCount, cooldownMs, now }) {
  if (prior == null) return true;
  const normalized =
    typeof prior === "number" ? { at: prior, count: 0 } : prior;

  const cooldownElapsed =
    cooldownMs > 0 && now - normalized.at > cooldownMs;
  const grewSinceLastPitch =
    currentCount >= normalized.count + REARM_GROWTH;

  return cooldownElapsed || grewSinceLastPitch;
}
```

```javascript
/**
 * Read user settings from chrome.storage.local. Defensive against missing or
 * malformed payloads â€” always returns a fully-populated settings object so
 * callers never have to null-check fields.
 */
async function getSettings() {
  try {
    const data = await chrome.storage.local.get("settings");
    const raw = data?.settings ?? {};
    const cooldownMs =
      typeof raw.consolidateCooldownMs === "number" && raw.consolidateCooldownMs >= 0
        ? raw.consolidateCooldownMs
        : DEFAULT_COOLDOWN_MS;
    return { consolidateCooldownMs: cooldownMs };
  } catch {
    return { consolidateCooldownMs: DEFAULT_COOLDOWN_MS };
  }
}
```

```javascript
/**
 * Tabs that share `hostname` with the navigating tab (excluding the navigator),
 * shaped to feed the consolidate interstitial's tab list. Sort:
 * most-recently-accessed first, then by window+index for stable grouping.
 */
async function findSameHostTabs(navigatingTabId, hostname) {
  const tabs = await chrome.tabs.query({});
  const out = [];
  for (const t of tabs) {
    if (t.id === navigatingTabId) continue;
    const u = tabUrlForPageMatch(t);
    if (!u || !isHttpUrl(u) || isExtensionUrl(u)) continue;
    let h;
    try {
      h = hostnameOf(u);
    } catch {
      continue;
    }
    if (h !== hostname) continue;
    out.push({
      id: t.id,
      windowId: t.windowId,
      index: t.index ?? 0,
      title: t.title || u,
      url: u,
      favIconUrl: t.favIconUrl || "",
      lastAccessed: t.lastAccessed ?? 0,
      pinned: !!t.pinned,
    });
  }
  out.sort((a, b) => {
    const la = b.lastAccessed - a.lastAccessed;
    if (la !== 0) return la;
    if (a.windowId !== b.windowId) return a.windowId - b.windowId;
    return a.index - b.index;
  });
  return out;
}
```

## Risk notes

- **Pitch payload shape change is read-tolerant, write-only-new.** v1.5 session data written as bare numbers reads fine via `evaluateConsolidationPitch`'s normalization, then gets overwritten the next time `recordConsolidationPitch` runs. No migration script needed.
- **`src/cooldown.mjs` is pure** â€” no chrome.* references â€” so the Node test is hermetic. If pure logic later needs to grow (e.g., per-host overrides in v2), keep it in this file and avoid pulling chrome.* in.
- **Chrome MV3 service-worker module imports** work for both `.js` and `.mjs` files when the SW is declared with `"type": "module"`. No manifest change needed for the new `cooldown.mjs` import.
- **`findSameHostTabs` includes pinned tabs.** Intentional â€” they're real tabs on the host. If pinning should exclude (debatable), filter `t.pinned` in the helper. Not in scope.
- **Tab list is captured at pitch time, not refreshed live.** If the user opens another ebay tab while staring at the interstitial, that new tab won't appear in the list. Acceptable for v1.6; revisit if it bites.
